ALB listener rules and per-route handlers are detailed in Deployed ALB Routing and Application Routing Rules below.

1. DNS app.example.com and api.example.com resolve through Route53 or Cloudflare.
2. ALB ALB terminates or receives edge traffic and applies path/method listener rules.
3. Compute Server ASG handles control/ingest; query ASG handles read-heavy query/event routes.
4. Durable Raw Path Server writes local parts, uploader sends to S3, S3 event notifies SQS, loader consumes.
5. Analytics Loader writes tenant-stamped rows to shared ClickHouse; query service reads with tenant filters.

In current Pulumi, the server ASG also runs nanotrace-loader beside nanotrace-server. That means ingest compute and loader compute are colocated today, while query compute is split into its own ASG.

Application Routing Rules

Route Public owner Behavior Tenant source
POST /events nanotrace-server gateway ALB default route sends ingest to the server target group. nanotrace-server validates ingest:write, resolves the organization, stamps tenant fields, and appends through EventLogWriter. API key organization or selected UI organization.
POST /query nanotrace-query ALB priority 10 sends public query traffic to the query target group. nanotrace-query validates query:read, allowlists table sources, injects tenant filters, and reads ClickHouse. Session/API key identity, inserted as __nanotrace_tenant_id.
GET /events/{event_id} nanotrace-query ALB priority 20 sends event detail reads to the query target group. The service looks up the ClickHouse event pointer with tenant filter, then reads the raw S3 byte range. Session/API key identity.
POST /internal/events Data-plane nanotrace-server Accepts only trusted gateway calls with internal secret. Appends tenant-stamped events to durable local parts. x-nanotrace-internal-organization-id.
POST /internal/query nanotrace-query Accepts trusted gateway calls. Allows only configured Nanotrace tables and rewrites sources to tenant-scoped subqueries. x-nanotrace-internal-organization-id.
GET /internal/events/{event_id} nanotrace-query Same event-detail lookup as public path, but internal-authenticated. x-nanotrace-internal-organization-id.
/auth/*, /organizations, /api-keys, /dashboards/*, /facets, /processors nanotrace-server control plane Control-plane APIs backed by Postgres and object storage metadata. Logged-in user membership or admin API key.
For the full deployed cluster, both ALB target groups are live: server ASG for control/ingest and query ASG for query/event-read. Internal routes are trusted service surfaces protected by x-nanotrace-internal-secret and x-nanotrace-internal-organization-id.

Control Plane Components

  • nanotrace-server full mode: UI, auth, sessions, orgs, invites, API keys.
  • Dashboard visualization persistence and frontend static serving.
  • Facet and processor config APIs.
  • Postgres stores control-plane state only; it can be managed RDS or an externally supplied Postgres URL, including a PrivateLink-reachable provider.

Data Plane Components

  • nanotrace-server owns ingest writes through POST /events and trusted POST /internal/events.
  • EventLogWriter writes local durable parts and uploader moves ready files to object storage.
  • nanotrace-loader consumes SQS notifications and loads ClickHouse.
  • nanotrace-query serves query and event-detail reads.

Isolation Controls

  • API keys and sessions resolve to exactly one active organization context per request.
  • Gateway forwards internal calls with organization id and shared secret.
  • Ingest stamps tenant_id and organization_id into events.
  • Read paths validate SQL sources and inject tenant filters into allowed tables.
